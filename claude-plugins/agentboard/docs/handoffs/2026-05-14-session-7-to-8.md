## READ THIS BEFORE TRUSTING THE design-reviewer PROFILE

The 9 reviewer rounds were **self-directed**: each reviewer subagent was
given specific scrutiny points written by the authoring agent (Claude).
This reproduced the author's blind spots across every round. The
finding-count "convergence" at rounds 7–9 (13 → 14 → 12) was reviewers
exhausting the directed scrutiny areas, **not** the profile converging on
correctness.

An independent Codex review (no scrutiny direction in the prompt) at
commit `47a1c14` then found Critical defects the directed rounds were
blind to — fundamental format mismatches between the reviewer profile and
the compose outputs it consumes:

- The profile assumed a "Requirements addressed" subsection and a
  "Rationale subsection" in L2/L3 `D#` entries. Neither exists — compose
  uses a five-part decision format (compose-l2 line 211, compose-l3 line
  316) with R#/Q# recorded in the decision body and alternatives in
  element 4. Fixed in `47a1c14`.
- The profile parsed contract annotations as `(consumed by ...)`. The
  real format is `— consumed by <card title(s)>` (em-dash). Fixed.
- L3 Scope heading handled as `## Scope (in / deferred / out)`; L3 uses
  plain `## Scope`. Fixed.
- Plan §7 R-DOC-4 used short bullet labels that don't match the §5
  schema. Fixed.

**These fixes are themselves unreviewed by an independent pass.** The
Codex review found the defects; the corrections in `47a1c14` have had no
independent review. Before this profile is trusted in the pipeline, run
ONE more independent review — fresh prompt, NO scrutiny direction, just:
"evaluate this profile against the plan and engineering standards, find
defects." Do not seed the reviewer with what to check.

## Methodology rule for all future sessions

Self-directed reviewer prompts reproduce author blind spots. When
dispatching a follow-up reviewer, brief it as a fresh independent review
of the work, never as a fix-verification and never with a list of
specific things to check. The Session 7 retrospective at
`docs/plans/2026-05-14-session-7-reviewer-pattern-analysis.md` documents
the round-by-round pattern analysis but was itself written under the
biased methodology — read it with that caveat.
