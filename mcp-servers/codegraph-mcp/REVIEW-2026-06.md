# codegraph-mcp — Deep Review & Remediation Plan (2026-06-08)

> Status: **review complete, no source changed yet.** This document is the
> durable record of a full-codebase audit and the phased plan to fix it.
> Scope: `mcp-servers/codegraph-mcp/` only.

## How this review was done

- **Baseline:** `npm install` clean, `tsc` builds with no errors, `npm test`
  = **148/148 passing**. So none of the findings below are caught by the
  existing suite — they are soundness, durability, and resolution-correctness
  gaps the tests don't exercise. Every fix should ship with a regression test.
- **Method:** the ~8,100 lines of `src/` were audited in four parallel passes
  (graph core + entry/cache/watch/export; the MCP tool layer; the tree-sitter
  pipeline + TS-compiler liveness; the per-language parsers). The highest-impact
  findings were then re-verified by hand against the source and the Node/TS
  APIs.
- **Standard:** the tool's own stated identity —
  *"deterministic, calibrated verdicts, never a false dead"* (`UPGRADE-PLAN.md`).
  The most serious findings are precisely where it breaks that promise.

Legend: **[verified]** = confirmed by hand during this review.
Counts: **1 Critical, 11 Serious, ~18 Moderate, ~14 Minor.**

## 🔴 Critical

### C1. A failed TypeScript Program reports the *entire* TS/JS codebase as dead — **[verified]**
`src/tscompiler/liveness.ts:76-81`. `covered` is populated with **all** TS/JS
files *before* the Program is built; the `if (!program) return { usedExternally,
covered }` early-out then returns that full `covered` set with an **empty**
`usedExternally`. `createTsProgram` returns `null` on any throw
(`liveness.ts:62-68`, cached). Result: every exported TS/JS symbol resolves to
`verdict:"unused"` and `find_dead_exports` reports the whole codebase dead. This
is fail-*dangerous*, the opposite of the core guarantee.
**Fix:** on the `!program` branch, `return { usedExternally, covered: new Set() }`
so `livenessFor` falls back to the safe syntactic verdict. Add a test that
forces a null Program and asserts no TS/JS symbol is reported dead.

## 🟠 Serious — soundness (false "dead" / wrong-file)

### S1. `export *` star-barrels produce false dead
`src/tscompiler/liveness.ts`. Uses are recorded only via identifier nodes;
`export * from './internal'` has no specifier node, so symbols exposed *only*
through a star-barrel (a standard public-API assembly pattern) are never marked
used → reported dead.
**Fix:** when visiting an `ExportDeclaration` with a module specifier and no
clause, resolve `checker.getExportsOfModule(moduleSymbol)` and mark each as
used (or propagate re-export so transitive deadness uses the true entry set).

### S2. `export default` is mis-handled (missed export or mis-keyed liveness)
`src/treesitter/symbols.ts:57-88` doesn't emit symbols for `export default
function/class`, and the consumer-side liveness key is `sym.getName()`
(`liveness.ts:97`), which is `"default"` for anonymous defaults vs the declared
local name elsewhere — the keys disagree.
**Fix:** emit default-export symbols with a stable key; key liveness on
`default` when the declaration carries `SymbolFlags.Default`. Fixtures for both
named-function-default and anonymous default.

### S3. Parse-error files silently degrade verdicts → false dead
`src/treesitter/analyze.ts` (+ all extractors). Nothing checks
`tree.rootNode.hasError`. A symbol whose only reference sits inside an `ERROR`
subtree (a mid-edit or grammar-gap file) gets reported dead.
**Fix:** when `rootNode.hasError`, exclude that file's symbols from *dead*
claims (treat as ambiguous) and surface a parse diagnostic.

### S4. `verify_doc` and `get_symbol` bypass the precision guards `find_dead_exports` enforces
`src/tools/query.ts:1262-1278` (`verify_doc`) and `:755-768` (`get_symbol`)
call `livenessFor` directly, without the `PRECISE_DEAD_LANGS` gate or the
method/FQN-type-only/C++-header/C++-ODR carve-outs that `find_dead_exports`
applies at `:833-859`. Two tools give contradictory verdicts for the same
symbol; name-resolved languages (e.g. Ruby) can be falsely "dead" in
`verify_doc`.
**Fix:** extract a single `deadVerdict(node, sym, ctx)` helper encapsulating all
carve-outs and have `find_dead_exports`, `get_symbol`, and `verify_doc` all call
it. (This one refactor closes S4 and M-ambiguous together.)

### S5. Go aliased same-name imports clobber resolution → false dead
`src/treesitter/connections.ts:281-284`. Registering symbols under the declared
package name overwrites a prior entry on a package-name collision, so
`pkg.Member` resolves into the wrong package; the right symbol is left with no
recorded user.
**Fix:** key on the import's alias/local name as authoritative; when two imports
would register the same key, mark ambiguous (register nothing) rather than
clobber.

### S6. Python absolute-import `rootDir` fallback resolves stdlib/third-party to a local same-named file
`src/parsers/python.ts:145-164`. The resolver always probes `searchBase` *and*
`rootDir`, so `import json` / `import requests` map to a local `json.py` /
`requests/__init__.py` anywhere under root — mislabeling external as internal
and fabricating edges.
**Fix:** resolve absolute imports only against the discovered package root, and
only when `parts[0]` is a real top-level package/module; drop the unconditional
`rootDir` fallback (or gate it so bare single-segment names aren't matched
against arbitrary files).

### S7. Python `from . import sibling` resolves to the package `__init__.py`, not `sibling.py`
`src/parsers/python.ts:112-143`. The imported name after `import` is discarded,
so `from . import b` in `pkg/a.py` edges to `pkg/__init__.py` instead of
`pkg/b.py`. Very common pattern → wrong-file edge.
**Fix:** capture the imported names and resolve each as a submodule of the
relative base (`name.py`, then `name/__init__.py`), falling back to the package
init only if the name isn't a submodule.

## 🟠 Serious — durability / concurrency (operational)

### S8. Non-atomic cache write — **[verified]**
`src/cache.ts:89`. `fs.writeFileSync` truncates then streams onto the live cache
file; a crash or a racing writer leaves a torn file that `JSON.parse` rejects on
next load, silently discarding the entire cache.
**Fix:** write to `file + "." + process.pid + ".tmp"` then `fs.renameSync` (atomic
on the same filesystem).

### S9. Watcher rescans race and clobber `currentGraph` — **[verified]**
`src/index.ts:931-940` (`rescanCurrentGraph`) + `src/watch.ts:58-68`. No
single-flight guard: overlapping debounced rescans (and a concurrent
`codegraph_scan`) read the same base graph and last-writer-wins, dropping events
and racing `saveCache`.
**Fix:** single-flight guard with a `pending`/dirty re-run flag; ideally also
serialize `codegraph_scan` against an in-flight rescan.

### S10. Shared tree-sitter `Parser` instance is not reentrant
`src/treesitter/parser.ts:59-83`. One cached `Parser` per grammar is handed to
every caller; the native `parse()` is stateful and non-reentrant, so any
overlapping/concurrent parse in this long-lived server corrupts trees.
**Fix:** cache the *language binding* (the expensive `setLanguage` input), not
the `Parser`; allocate a fresh `Parser` per `parseSource` (cheap) or use a small
checkout pool.

### S11. Recursive `fs.watch` is unsupported on Linux below Node 20, but `engines` allows Node 18 — **[verified]**
`src/watch.ts:48` + `package.json` `engines.node: ">=18.0.0"`. On Node 18 +
Linux, `fs.watch(dir, { recursive: true })` throws
`ERR_FEATURE_UNAVAILABLE_ON_PLATFORM`; the watch feature can never work on the
declared minimum. (Works here only because the box runs Node 22.)
**Fix:** raise `engines.node` to `>=20` and document it, or detect the platform
and fall back / emit a clear, cause-naming error.

## 🟡 Moderate

### Resolution correctness
- **M1. Comment-stripping is inconsistent.** JS/C++ strip comments before
  matching, but **Go** (`golang.ts:78`), **Rust** (`rust.ts:49`), and **go.mod**
  (`gomod.ts:33`) do not → commented-out imports / `replace` directives become
  real edges. Fix: strip comments uniformly across all regex parsers.
- **M2. Phantom edges from missing existence checks.** Ruby `require_relative`
  (`ruby.ts:23`) and requirements `-r` targets (`requirementstxt.ts:31`) add
  edges without `existsSync`; Go in-module paths are classified `internal`
  without a directory check (`golang.ts:40` → `edges.ts:68`), so a typo'd
  internal import looks healthy. Fix: stat before edging; make Go resolution
  tri-state.
- **M3. tsconfig block comments break alias resolution.** `javascript.ts:31`
  strips only `//`; a `/* */` in a JSONC tsconfig makes `JSON.parse` throw →
  `loadTsConfig` returns null → *all* `@/...` imports misclassified external.
  Fix: strip block comments + trailing commas, or use a JSONC parser; also
  follow `extends` (m-level).
- **M4. C++ resolves to the first same-named header across all dirs.**
  `cpp.ts:55-71` searches every header-bearing directory and returns the first
  hit → wrong `config.h`. Fix: restrict to the including file's dir + configured
  include roots; report ambiguity instead of guessing.
- **M5. Python package-root detection mishandles PEP 420 namespace packages**
  (`python.ts:99-110`) → returns `rootDir` and over-resolves via S6.

### Performance (the "large repo" claim)
- **M6. O(n) `Array.shift()` BFS everywhere** — **[verified]** —
  `graph.ts:659,749,789,822,916,1226`. Every traversal dequeues with `shift()`
  (O(n)), making each BFS O(V²). Fix: head-index pointer or a deque.
- **M7. `get_symbol` is O(K·N·E)** (`query.ts:752-770` calling `symbolImporters`
  per matching definition). Fix: build a `Map<targetFile, ImportEdge[]>` once per
  call (or memoize on the graph).
- **M8. `find_unused_imports` re-reads files from disk per call**
  (`query.ts:903-911`) → staleness vs the graph + uncached parse cost. Fix:
  compute during scan and store on `FileNode`.
- **M9. Doc-scan compiles O(docs×nodes) regexes** (`graph.ts:515` via
  `scanDocForCodeReferences`). Fix: single combined scan / Aho-Corasick.
- **M10. Parsers double-stat (`existsSync`+`statSync`) and don't cache the Python
  package root** (`javascript.ts`/`python.ts`). Fix: one `statSync` in try/catch;
  memoize package root per directory.

### Correctness of output / consistency
- **M11. `ambiguousCount` is permanently 0** (`query.ts:862`) — `livenessFor`
  never returns `"ambiguous"`, yet the tool advertises the bucket as
  load-bearing (`index.ts:1151`). Fix: actually produce `ambiguous` for the
  documented namespace/dynamic/star cases on non-covered languages, or remove
  the field and correct the docs. (Folds into the S4 shared-helper refactor.)
- **M12. Incremental vs full-scan divergence.** `removed` count via subtraction
  can be wrong/negative (`graph.ts:418`; use a set difference); manifest renames
  leave stale edges on reused importers (`graph.ts:447-451`); FQN edges may
  differ. Fix the count; document/test the divergence cases.
- **M13. Paired tools don't round-trip.** `get_external_users` vs
  `list_external_dependencies` use different name bucketing (`query.ts:542` vs
  `:488`); a scoped package queried by short name returns nothing. Fix:
  normalize both through `externalName`.
- **M14. `orderCycle` can report an incomplete cycle ring** — **[verified by
  reading]** — `graph.ts:1058`. A greedy intra-SCC walk dead-ends and silently
  drops members, so `find_cycles` understates a real cycle. Fix: assert
  `ring.length === scc.length`; reconstruct a true ring (the SCC itself is
  correct — only the display ordering is lossy).
- **M15. Path-traversal / arbitrary-read surface.** `root_dir` is not validated
  (`index.ts:177`); scans read any file the caller points at into responses/
  cache (`graph.ts`). Fix: validate root exists & is a directory; refuse `/` /
  `$HOME`; document that contents are read.
- **M16. Export language-filter drops edges invisibly** (`export.ts:74-118`) —
  only node-cap truncation is surfaced. Fix: report dropped-edge count.
- **M17. `cache` doesn't verify `rootDir` on load** (`cache.ts:69-84`) — a shared
  cache dir or a hashing change adopts a foreign graph. Fix:
  `if (path.resolve(rootDir) !== graph.rootDir) return null`.
- **M18. Cache key path portability** (`cache.ts:44-50`) — raw OS-path hash;
  Windows drive/case variants produce different keys. Fix: normalize before
  hashing or document per-OS.

## ✅ What's solid (do not touch)

The hard graph algorithms are correct: iterative Tarjan SCC, SCC-condensation +
Kahn layering on cyclic graphs, cycle normalization, self-loop/empty-graph
handling. The soundness *discipline* is real (name-resolved languages gated out
of dead-code, methods excluded, documented "safe direction" reasoning). The
findings above are leaks in an otherwise principled design.

## ❌ Rejected finding (recorded so it isn't re-raised)

An auditor claimed the watcher (`watch.ts:58`) misses file adds/deletes by
listening for `"change"` not `"rename"`. **False.** Node's `FSWatcher` emits the
event *named* `'change'` for both `eventType` values; `rename` (create/delete)
*does* fire the handler. The comment at `watch.ts:57` is correct and a passing
test exercises it.

## Remediation plan (phased)

Each phase is an independent, shippable unit with its own tests. Order is by
value and by dependency (shared refactors first within a phase).

### Phase 1 — Restore the "never a false dead" guarantee (soundness)
The tool's identity. Do this first.
1. **C1** — empty `covered` on null Program (one line + test). *Start here.*
2. **S4 + M11** — extract a single `deadVerdict()` helper; route
   `find_dead_exports`, `get_symbol`, `verify_doc` through it; make `ambiguous`
   real (or remove it). This is the structural keystone — several findings
   collapse into it.
3. **S3** — `hasError` guard: errored files can't downgrade a symbol to dead.
4. **S1** — resolve `export *` targets in the liveness pass.
5. **S2** — handle `export default` (emit + key consistently).
6. **S5** — Go aliased same-name imports: alias-key, ambiguous-on-collision.
- *Exit:* a new soundness test file covering each (null-program, star-barrel,
  default export, parse-error, Go alias collision) all green; existing 148 green.

### Phase 2 — Resolution correctness (right file, no phantoms)
7. **S6, S7** — Python absolute-fallback + `from . import x`.
8. **M2** — existence checks: Ruby, requirements `-r`, Go tri-state.
9. **M1** — uniform comment stripping (Go, Rust, go.mod).
10. **M3** — tsconfig JSONC (block comments / trailing commas / `extends`).
11. **M4** — C++ scoped include search.
12. **M5** — Python namespace-package root.
- *Exit:* per-language resolution fixtures (wrong-file + phantom-edge cases).

### Phase 3 — Durability & concurrency (trust it to run unattended)
13. **S8** — atomic cache write (temp + rename).
14. **S9** — single-flight watcher rescans; serialize against `scan`.
15. **S10** — per-call parser, cached language binding.
16. **S11** — `engines.node >=20` (+ platform note) or watch fallback.
17. **M17, M18, m11** — cache rootDir verification, key portability, signal-safe
    exit.
- *Exit:* concurrency test (overlapping rescans converge); torn-write recovery
  test.

### Phase 4 — Performance at scale
18. **M6** — replace all `Array.shift()` BFS with head-index/deque.
19. **M7, M8, M9, M10** — index `symbolImporters`; precompute unused-imports;
    combined doc scan; single-stat + cached package root.
- *Exit:* a benchmark fixture (synthetic N≈10k-node graph) with before/after.

### Phase 5 — Output consistency, API polish, minors
20. **M12** (incremental `removed` set-difference + documented divergence),
    **M13** (external-users bucketing), **M14** (cycle ring completeness),
    **M15** (root_dir validation), **M16** (export edge-drop reporting), then the
    `m*` minors.
- *Exit:* full suite green; docs (`README.md`) reconciled with actual behavior
  (notably the `ambiguous` story and watch platform support).

### Standing test discipline
Every fix lands with a regression test that **fails before, passes after** —
the existing suite is green today and missed all 40 findings, so behavior-level
tests are the point, not coverage theater.
