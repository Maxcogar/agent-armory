  ## What changed (so you know what to look for)

  Old version (≤ commit `d7f3161`) had 6 tools: `rag_setup`, `rag_index`,
  `rag_check_constraints`, `rag_query_impact`, `rag_health_check`, `rag_status`.
  It wrote `.rag/config.json`, `.rag/collections/`, `ARCHITECTURE.yml`, and
  `docs/patterns/` into each project's tree.

  New version has 2 tools: `rag_search`, `rag_query_impact`. It auto-detects
  project root, caches under `~/.cache/codebase-rag/<sha1[:16]>/` (on Windows:
  `%LOCALAPPDATA%\codebase-rag\<hash>\`), and runs a live filesystem watcher.
  Nothing is written into the project tree.

  The MCP config shape is the same (`command: python`, `args: [server.py path]`).
